## Claude:
You're right to call this out — root cause is a race between two async operations at the WindowServer:

1. Cross-app focus change (Bear → Chrome) is async
2. `minimize()` on Bear's frontmost window triggers macOS's own z-order recalc

Whichever lands first at WindowServer wins. We've been reordering them in the *Lua* code, but at the *system* level both are queued back-to-back, so it's a coin flip.

**Real fix:** focus Chrome, *wait for it to settle*, then minimize. By the time minimize fires, the note is no longer in the frontmost app, so its minimize doesn't trigger Bear z-order behavior. Let me also strip the debug prints since we know what's happening.
