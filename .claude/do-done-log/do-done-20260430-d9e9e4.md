## Eli:
let's tweak the way that L010 shoving/stretching works. I find myself almost always annoyed by stretching, it's almost never what I want, I want to move not stretch! And if I do want to stretch we already have that with the snapped-edge stepResize. So let's stop stretching back. Funny enough if we carried this fully this would imply that the whole persisted virtual frames approach wasn't needed and all I need is a pure mechanical shoving to shrink on moving when at the edge. Let's refactor down to that, we already did the work and it's committed (right?), so if we ever want to get back to it, the work is still there. It was a complex abstraction that got some careful turns to get right and we should keep any code improvements that came from it but drastically pare down it to the more mechanical simpler version.
